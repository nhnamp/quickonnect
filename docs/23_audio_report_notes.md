# Audio Report Notes

## What Was Done
Created and then simplified a Vietnamese report file explaining the audio feature in QuicKonNect. The current report focuses on what technologies are used, why they were chosen, how audio data flows through the network, what happens with two machines, how ngrok changes the connection path, and likely defense questions with suggested answers.

## Why It Matters
Audio is one of the most technically important features in the project because it demonstrates continuous TCP streaming, multithreaded capture/playback, server-side processing, encrypted transport, and optional speech-to-text. A focused report helps the team explain both the implementation and its trade-offs during the project defense.
